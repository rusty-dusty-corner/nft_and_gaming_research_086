## 5. Architecture – Verifying NFT Ownership  

### 5.1 High‑Level Flow  

```
+----------------+       1. Connect Wallet (WalletConnect)      +----------------+
|   Mobile UI    | ------------------------------------------> |   Wallet App   |
+----------------+                                            +----------------+
        ^                                                          |
        | 2. Sign nonce (on‑device)                               |
        |                                                          v
+----------------+       3. Send (address, signature, nonce)   +----------------+
|   Game Client  | ----------------------------------------> |   Backend API |
+----------------+                                            +----------------+
        ^                                                          |
        | 4. Verify signature & query blockchain (Alchemy)         |
        |                                                          v
+----------------+       5. Return JWT with NFT payload       +----------------+
|   Backend API  | <---------------------------------------- |   Blockchain   |
+----------------+                                            +----------------+
        ^                                                          |
        | 6. Store JWT locally (secure storage)                     |
        v                                                          v
+----------------+       7. Attach JWT to future IAP / game   +----------------+
|   Game Client  | <---------------------------------------- |   Game Server |
+----------------+                                            +----------------+
```

### 5.2 Technical Details  

| Component | Tech Stack | Key Functions |
|-----------|------------|---------------|
| **Auth UI** | Unity UI + WalletConnect SDK | Prompt user, display QR / deep‑link, request connection. |
| **Signature** | `eth_personalSign(message, address)` | One‑time nonce (UUID + timestamp) prevents replay attacks. |
| **Backend** | Node.js / Express + **ethers.js** | Verify signature, call `balanceOf` / `tokenOfOwnerByIndex` / custom `tokenType` view. |
| **Cache** | Redis (TTL 300 s) | Store `address → ownedNFTs` mapping to avoid repeated RPC calls. |
| **JWT** | HS256 (secret stored in Vault) | Payload: `{ address, owned: [{id, type}], iat, exp }`. |
| **Unity** | `UnityWebRequest` + **JWTManager** script | Attach JWT to all HTTP calls (IAP validation, bonus activation). |
| **Server‑side IAP validation** | Verify Google/Apple receipt **AND** apply NFT discounts based on JWT. | Guarantees that discounts cannot be spoofed client‑side. |

#### Sample Backend Snippet (Node.js)

```js
const { ethers } = require('ethers');
const provider = new ethers.providers.AlchemyProvider('matic', process.env.ALCHEMY_KEY);
const nft = new ethers.Contract(process.env.NFT_ADDRESS, NFT_ABI, provider);
const redis = require('redis').createClient();

async function getOwnedNFTs(address) {
  const cached = await redis.get(address);
  if (cached) return JSON.parse(cached);

  const balance = await nft.balanceOf(address);
  const ids = [];
  for (let i = 0; i < balance; i++) {
    const tokenId = await nft.tokenOfOwnerByIndex(address, i);
    const type = await nft.tokenType(tokenId); // custom view returns 'A'|'B'|'C'|'D'
    ids.push({ tokenId: tokenId.toString(), type });
  }
  await redis.setEx(address, 300, JSON.stringify(ids));
  return ids;
}
```

#### Unity Example (C#)

```csharp
public IEnumerator Authenticate(string address, string signature, string nonce) {
    string url = $"{BASE_URL}/auth?addr={address}&sig={signature}&nonce={nonce}";
    using (UnityWebRequest request = UnityWebRequest.Get(url)) {
        yield return request.SendWebRequest();
        if (request.result != UnityWebRequest.Result.Success) {
            Debug.LogError($"Auth failed: {request.error}");
            yield break;
        }
        string jwt = request.downloadHandler.text;
        PlayerPrefs.SetString("GAME_JWT", jwt);
        Debug.Log("Auth successful, JWT stored.");
    }
}
```

## 11. Legal & Regulatory Safeguards  

| Issue | Solution |
|-------|----------|
| **NFT classification** | Explicitly marketed as **utility** (discounts, in‑game bonuses) – no claim of profit‑sharing or equity. |
| **KYC/AML** | Required only for purchases > € 10 k (private round) and for players who cash‑out Gold‑tokens to fiat. Integrated via **Jumio** API. |
| **GDPR** | Store only wallet address & minimal email; encrypt data at rest; provide right‑to‑erasure via support ticket. |
| **Age rating** | Game rated **12+**; NFT purchases gated by age verification (self‑asserted DOB + parental consent for minors). |
| **Intellectual property** | All NFT artwork licensed under **CC‑BY‑NC‑SA** for in‑game display; resale rights retained by owners. |
| **Tax compliance** | Partner with a crypto‑tax advisory firm to generate user‑friendly transaction reports (for EU & US). |

## 12. Risk Matrix & Mitigation  

| Risk | Probability | Impact | Mitigation |
|------|-------------|--------|------------|
| **Regulatory crackdown on NFTs** | Medium | High (could force removal) | Design NFTs as *utility*; keep ability to disable discounts via server flag; maintain a “pure” free‑to‑play mode. |
| **Network congestion** | Low (Polygon) | Medium (delayed mint/transfer) | Use **meta‑transactions** (gas‑less for users) and batch minting for bulk drops. |
| **Pay‑to‑win perception** | Medium | High (player churn) | Cap discounts (max 30 %); balance via skill‑based PvP matchmaking; regular free‑to‑play events. |
| **Smart‑contract bugs** | Low (audit) | High (fund loss) | Dual‑audit + bug‑bounty program (minimum € 5 k). |
| **Low secondary‑market liquidity** | Medium | Medium | List NFTs on **OpenSea**, **Immutable X Marketplace**, and build an in‑game auction house. |
| **Server overload on auth spikes** | Low | Medium | Autoscaling on AWS (Lambda + API Gateway) + Redis cache; load‑test with 10 k concurrent auths. |
